SCR-05

크롤링 버튼 상태 전이

크롤링 버튼은 4가지 상태를 가진다. 비동기 백그라운드 실행 + 2초 폴링으로 UI를 업데이트한다.

1 대기 상태 (기본값)

클릭 가능. POST /api/crawl 요청 전송.

2 실행 중 (비활성화)
크롤링 시작 — 완료 후 자동 갱신

GET /api/crawl/status 2초마다 폴링 → 진행 메시지를 버튼 텍스트에 반영.

3 중복 요청 차단
⚠️ 이미 크롤링 중입니다

실행 중 재요청 시 서버가 "status":"running" 반환 → 토스트만 표시.

4 완료 & 자동 갱신
✅ 크롤링 완료 — 대시보드 갱신됨

status='완료' 수신 → 버튼 복구 → 완료 토스트 → 전 섹션 데이터 reload.

폴링 흐름: POST /api/crawl → (200 started) → setTimeout 루프: GET /api/crawl/status 2초마다 → running=false && message='완료' → 갱신.
오류 상태: message가 '오류:'로 시작하면 빨간 토스트 + 버튼 복구. 재시도 안내 표시 권장.
토스트 소멸: 완료·오류 토스트는 3초 후 자동 사라짐 (setTimeout + CSS opacity transition).

UI-03 섹션 탭 순위 / 키워드 / 비교 3개 탭 전환. 활성 탭: 보라 언더라인(#534AB7) + 폰트 볼드. — Must
UI-04 베스트셀러 카드 표지 이미지 + 순위 뱃지 + 제목 + 저자 표시. 분야·서점 필터 조합으로 카드 갱신. 이미지 오류 시 회색 플레이스홀더. GET /api/books Must
UI-05 키워드 분야 선택 <select> 드롭다운. 변경 시 Chart.js 차트 즉시 갱신. 기본값: 첫 번째 분야. GET /api/trends Must
UI-06 키워드 막대 차트 Chart.js 수평 막대 차트. 분야별 Top 10 키워드 빈도. 빈도 수치 막대 끝 레이블 표시. GET /api/trends Must
UI-07 서점별 비교 패널 분야별 알라딘/Yes24 좌우 2열 배치. 9개 분야 스크롤 표시. 공통 도서 초록 하이라이트. GET /api/compare Must
UI-08 크롤링 토스트 시작·진행·완료·오류 4가지 상태 메시지. 완료/오류 토스트 3초 후 자동 소멸. GET /api/crawl/status Must
UI-09 공통 도서 배지 두 서점 동시 등장 도서에 초록 "공통" 배지 + 초록 배경(#EAF3DE). 베스트셀러 카드 · 비교 패널 양쪽에 표시. GET /api/compare Must
UI-10 빈 상태 처리 DB 데이터 없을 시 "🔄 지금 크롤링 버튼을 눌러 데이터를 수집해 주세요" 안내 표시. — Should
UI-11 반응형 레이아웃 화면 폭 768px 이하: 책 카드 1열, 비교 패널 1열로 전환. — Nice
